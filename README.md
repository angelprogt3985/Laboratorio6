# Parte 1 - Laboratorio 6 Angel Camargo - 24003664

Para esta primera parte se requería crear un sistema de favoritos, en donde las personas puedan administrara las recetas que más les gustaron, añadiendo una pestaña exclusiva para que ellas puedan ver las recetas que más le gustaron, igual que marcar y desmarcarlas de favoritos según ellos requieran, primeramente se necesito de configurar toda la parte de firebase para poder administrar toda la parte de Gemini y analizar la imagen por IA. 

Después de esto se hicieron cambios en el domain/model/Recipe.kt donde añadimos un nuevo parametro isFavorite, que se encarga de administrar dependiendo de si la persona le dio favorito a la recete o no, se pone en true o false según corresponda, después de esto se modifico el data/firebase/FirestoreRepository.kt en donde añadimos la función toggleFavorite, la cuál se encarga de actualizar en Firestore si una receta es favorita o no, procedente a esta parte actualizamos ui/viewmodel/ChefViewModel.kt en donde nos encargamos de conectar la UI con Firestore para marcar o desmarcar una receta como favorita, por último se actualizo el archivo ui/screens/RecipeListScreen.kt, en cada receta que apareciera se añadió el boton de corazón para poder marcarlo o desmarcarlo si el usuario quisiera añadir como favorito dicha recete, igualmente se creo el boton ver Favoritas donde el usuario puede ver exclusivamente las recetes que marco como favoritas para poder tener un mejor orden al momento de la visualización.

# Parte 2 - Laboratorio 6 Angel Camargo - 24003664

Para esta parte se necesitaba crear un sistema por el cuál el usuario se le permitía compartir la recete por medio de otras apps, como lo serían Facebook, Whatsapp entre otras, de tal manera que se tomaba una foto de toda la receta guardandola por medio de un archivo png para poder compartirla de una manera más fácil y más entendible a muchas más personas según el usuario lo requiera.

Para lograr esto primeramente creamos el archivo util/ShareUtils.kt el cual se trata de un objeto utilitario que convierte un Bitmap en archivo .png temporal en el caché del dispositivo, igualmente nos ayuda a poder generar una URI segura con FileProvider, lanza el share sheet nativo de Android con Intent.ACTION_SEND, y para administrar de una mejor forma el espacio, se elimina el archivo temporal después de 60 segundos. Después de haber creado dicho archivo, creamos en res/xml, file_paths.xml el cual se encarga de definir la ruta de caché permitida para que FileProvider pueda compartir archivos con otras apps, en el AndroidManifest se registró el FileProvider de AndroidX para permitir compartir imágenes de forma segura con otras apps y finalmente en ui/screens/RecipeDetailScreen.kt, agregamos en el FAB un ícono de compartir, en donde el usuario lo puede presionar y sale el menu para poder compartir la receta, se integró graphicsLayer para capturar el contenido de la pantalla como Bitmap, igualmente se agrego un estado de carga el cuál muestra un CircularProgressIndicator mientras la imagen se prepara y finalmente se integró con ShareUtils para lanzar el share sheet.
